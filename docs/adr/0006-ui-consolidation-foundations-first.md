# UI Consolidation — Foundations-First, Audit + Design.md Aligned

The codebase carries two overlapping UI debts: `AUDIT_FULL.md` documents ~35 current-state defects (raw hardcoded colors, emoji-as-icons, missing table pagination, a broken conflict-resolution button, native `<select>` in `kid-form.tsx`), and `DESIGN.md` specifies the future-state brand system (warm cream canvas, four-tier green, 50px pill buttons, `scale(0.95)` active, Hugeicons). We rejected treating these as separate efforts because the audit's single highest-impact finding — pervasive raw Tailwind colors (`text-zinc-500` ×119, `border-zinc-200` ×58, status greens/ambers/reds with no semantic home) — is the exact prerequisite the DESIGN.md theme needs to adopt. Doing the audit fixes first, then re-skinning later, would touch every file twice.

We also rejected full DESIGN.md realization. Most of DESIGN.md was extracted from the marketing site (`littlerabbani.com`) — feature bands, hero photography, cookie consent, Lora/Caveat typefaces, decorative blobs, the WhatsApp floating button. Those are marketing-page concerns, not back-office LMS concerns. The dashboard-relevant subset is the _tokens_: color tiers, cream canvas, pill radius, the layered shadow scale. So DESIGN.md is honored at the token/adoption layer and its marketing-only elements are deferred. The floating WhatsApp button already exists (`src/components/layout/whatsapp-button.tsx`); its presence is an existing decision, not new work.

**Decision — execute foundations-first, bugs front-loaded, in four tracks:**

1. **Bugs first (local, unblock users):** Fix the C-7 conflict dialog (`"Simpan Keduanya"` sends server values instead of local — add `localFields` to `IConflictData`, populate it, submit it; kill the silent `catch {}`, add toast + loading spinner). Fix C-5 (native `<select>` in `kid-form.tsx:130-145` → shadcn `Select`, pattern exists 60 lines below it).
2. **Token foundation:** Add `--success`/`--success-foreground`/`--warning`/`--warning-foreground` to `globals.css` `:root` + `.dark`, mapped from DESIGN.md (success → brand-mint family; warning → brand-gold `#EAB308`), exposed via `@theme inline`. Add a global Button pill override (`rounded-full`) so DESIGN.md's universal 50px pill holds without editing `src/components/ui/` (AGENTS.md forbids it).
3. **C-2 sweep:** One exhaustive pass. Capture a raw-color → semantic-token mapping table first, then apply file-by-file. Fold C-4 (adopt the orphaned shared `Pagination`/`SearchInput`/`EmptyState` components where a file duplicates them inline), C-3 (structural emoji → Hugeicons; mood-as-data-content is acceptable), and C-9 (remove `h-4 w-4` from Button-child icons — they override the `size-4` variant system; add `data-icon="inline-start"`/`inline-end` so the Button's `has-data-[icon*]` padding selectors fire) into each file as it is visited — one touch per file, no re-churn.
4. **C-1 remainder:** Only the genuinely-unpaginated screens — monthly & quarterly report pickers, teacher session picker, daily-report kid list — adopt shared `Pagination` + `SearchInput`. The owner kid/guardian/activity list pages already paginate inline and are left untouched (AGENTS.md: no refactoring working legacy code).

**Verification:** `bun typecheck` + `bun lint` after each foundation step; `bun build` after the full sweep; one regression test for the C-7 keep-local-values path (the only non-trivial logic change); visual sample of a couple pages in dev. The mechanical sweep itself gets no per-site test — that would be pre-hardening the happy path.

**Tradeoffs.** Foundations-first means no user-visible prettifying until the sweep lands — early sessions look like "nothing changed." We accept that because the alternative (pixel-fixing against the wrong tokens) is pure rework. The single-sweep-with-fold-ins approach concentrates risk in one large mechanical pass rather than spreading it, but keeps churn to one touch per file and lands the theme foundation once. Scoping C-1 to only the broken pickers leaves the working owner pagination inconsistent (inline vs. shared component) — accepted as deliberate, per the no-refactor-working-code rule; a future consistency pass can unify if it becomes a maintenance problem.

**Scope corrections from verification** (recorded so the plan runs from accurate scope, not the stale audit synthesis): C-8 (teacher tab bar covering content) was already shipped at `teacher/layout.tsx:25` (`pb-20 md:pb-0`) — removed. C-1 owner list pages were already paginated inline in the `shadcn-compliance` commit — leaves only the report pickers + teacher/daily lists. C-2 is ~600 raw-color sites across ~50 files (not "~50 files" vaguely). The live critical set is 7, not 9.
