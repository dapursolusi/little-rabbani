# Capture-First Architecture

The LMS's primary job is to capture observations; all reports (Daily Parent, Monthly, Quarterly) are generated views of stored data. We rejected the alternative of a faster per-report writer UI because it would leave the root cause unaddressed — the wife's monthly/quarterly reconstruction pain exists *because* daily data isn't persisted, not because report-writing is slow. Capture-first also makes the two-session plan viable: distributed capture by teachers becomes a 10-second tap rather than a 30-minute composition, dissolving the salary constraint that today halts reports entirely.
